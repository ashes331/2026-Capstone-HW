## 1. Claude AI 개요

### 1.1 배경

| 항목 | 내용 |
|---|---|
| 개발사 | Anthropic (2021년 설립) |
| 창립자 | Dario Amodei (OpenAI 전 부사장) |
| 이름 유래 | Claude Shannon — 정보 이론의 아버지 |
| 핵심 철학 | "안전하고 정확한 AI" |

### 1.2 주요 특징

- **윤리적 설계**: 안전하고 책임 있는 응답 지향
- **Artifacts**: 대화 중 코드·문서·웹사이트 실시간 생성 및 미리보기
- **200K 토큰 컨텍스트**: 소설 한 권 분량(~15만 단어)을 한 번에 처리
- **Hallucination 최소화**: 모르는 것은 모른다고 명시
- **Extended Thinking**: 복잡한 문제를 단계별로 사고하는 심층 추론 기능

## 3. 주요 기능

### 3.1 설정 (개인 맞춤)

`설정 → 일반 → 개인맞춤설정`에서 Claude의 응답 스타일을 고정 지정 가능.

```
신원 / 역할 — 인공지능소프트웨어학과 학생
전문 분야 — ML/DL, NLP, 프로그래밍
주요 사용 목적 — 코드 작성, 논문 검토
응답 스타일 — 학술적·정확한 표현, 기초 설명 생략
```

### 3.2 프로젝트

관련 대화를 하나의 작업 공간으로 묶어 관리. 무료 사용자는 최대 5개 생성.

| 구성 요소 | 설명 |
|---|---|
| **Persistent Context** | 프로젝트 내 모든 대화에서 공유되는 System Prompt |
| **파일 업로드** | PDF·코드·문서를 프로젝트 레벨에서 공유 → 매번 재업로드 불필요 |
| **대화 분류** | 주제별 대화 관리 및 이전 대화 재참조 |

#### 프로젝트 지침 활용 시점
- 프로젝트별 고유 맥락 제공
- 특정 워크플로 규칙 정의
- Claude의 역할·관점 고정

### 3.3 Artifacts

Claude가 대화창 옆 별도 패널에 독립적으로 렌더링하는 결과물 생성 기능.

| 타입 | 확장자 | 용도 |
|---|---|---|
| Markdown | `.md` | 보고서, 문서, 강의 노트 |
| HTML | `.html` | 웹 페이지, 인터랙티브 UI |
| React | `.jsx` | 동적 컴포넌트, 대시보드 |
| SVG | `.svg` | 다이어그램, 일러스트 |
| Mermaid | `.mermaid` | 플로우차트, ER 다이어그램 |
| PDF | `.pdf` | 인쇄용 문서 |

#### 핵심 특징
- **렌더링 패널 분리**: 코드 설명과 결과를 좌우로 동시 확인
- **반복 수정(Iterative Editing)**: "색상 바꿔줘" 같은 후속 요청으로 점진적 개선
- **다운로드 및 복사**: 결과물 즉시 저장 가능

#### JSX 활용 예시
```
React 기반 인터랙티브 UI — recharts 기반 데이터 시각화,
계산기, 대시보드, 학생 실습용 인터페이스 등
```

#### Mermaid 지원 다이어그램 종류
- `flowchart` — 알고리즘 흐름, 의사결정
- `sequenceDiagram` — API 통신, 클라이언트-서버
- `classDiagram` — OOP 설계, UML
- `erDiagram` — DB 스키마
- `stateDiagram-v2` — 오토마타, 상태 머신
- `gitGraph` — 브랜치 전략

## 4. 스킬(Skill) 시스템

### 4.1 개념

> **스킬 = 반복 가능한 작업을 위한 구조화된 지침을 미리 등록해 자동으로 적용하는 기능**

- 특정 업무의 SOP(표준 운영 절차)를 Claude 컨텍스트에 주입
- 좋은 프롬프트를 매번 재작성하는 비용 제거
- 일관된 출력 품질 유지

### 4.2 작동 원리

```
대화 시작
  └─ 스킬 메타데이터(name, description)만 먼저 로드
        └─ 관련 요청 감지 시 전체 SKILL.md 본문 로드 → 실행
```

- **점진적 로딩**: 컨텍스트 과부하 방지, 활성화된 스킬만 선택 사용
- **자연어 정의**: 코딩 없이 커스텀 스킬 생성 가능

### 4.3 SKILL.md 구조 (핵심 파일)

```markdown
---
name: skill-name           # 스킬 식별자 (항상 메모리 상주)
description: |             # 트리거 조건 + 수행 기능 요약
  언제 이 스킬을 써야 하는지,
  무엇을 하는 스킬인지 선언적으로 기술
---

# 스킬 제목

## 전체 프로세스
- 단계 1
- 단계 2

## 주요 규칙
- 규칙 A
- 규칙 B

## 출력 형식
...
```

> **핵심 원칙**: description은 자동 트리거 판단 기준 → 명확한 트리거 키워드 포함 필수

### 4.4 폴더 구조

```
~/.claude/skills/
└── skill-name/
    ├── SKILL.md          # 필수: YAML + 본문
    ├── references/       # 선택: 상세 참조 문서 (분량 제한 없음)
    ├── scripts/          # 선택: 자동화 스크립트, eval 코드
    └── assets/           # 선택: 템플릿, 폰트, 아이콘
```

### 4.5 기본 제공 스킬 목록

| 스킬 이름 | 설명 | 활용 예 |
|---|---|---|
| `skill-creator` | 스킬 생성·개선·성능 측정 | 커스텀 기능 개발 |
| `canvas-design` | PNG·PDF 시각 디자인 | 포스터, 보고서 |
| `web-artifacts-builder` | HTML 복잡 웹 결과물 | 대시보드, 웹 UI |
| `algorithmic-art` | p5.js 생성형 아트 | 인터랙티브 그래픽 |
| `mcp-builder` | MCP 서버 구축 지원 | 외부 API 연동 |
| `doc-coauthoring` | 구조화 문서 공동 작성 | 논문, 기획서 협업 |
| `paper-writing` | 학술 논문 작성 지원 | KCI·IEEE 논문 |
| `theme-factory` | 결과물 테마 적용 | 슬라이드, 웹 스타일링 |

## 5. 스킬 활용 실습

### 5.1 스킬 생성 절차 (8단계)

```
1. 반복 작업 선정  →  반복성·규칙성·일관성 있는 작업
2. 요청 프롬프트 작성  →  스킬 목적 기술 + Claude에게 프롬프트 정제 요청
3. Claude의 질의 응답  →  객관식 질의(비중, 시장 범위 등)에 응답
4. 스킬 생성 대기  →  백그라운드 작업 과정 확인
5. 생성 결과 검토  →  description 트리거 키워드 확인, 본문 규칙·예시 확인
6. 수정 요청  →  필요 시 구체적 수정 사항 전달
7. 테스트  →  자연스러운 문장으로 스킬 호출
8. 반복 개선
```

### 5.2 좋은 프롬프트 작성법

나쁜 예시:
```
주식과 ETF 추천해 줘
```

개선된 예시 (스킬용):
```
최근 주식 시황을 분석하여 유망 섹터를 선정하고,
해당 섹터 수혜 예상 국내·미국 ETF 및 개별 주식 종목을
투자 기간(단기·중기·장기)별로 각각 5개씩 추천해 주세요.
분석 기준: 최신 글로벌·국내 거시경제 지표, 실적 발표, 산업 트렌드
ETF 선정 기준: 과거 수익률 우수 + 보수율 낮은 순
```

> **핵심**: 목적·기준·출력 형식·분류 체계를 명시할수록 스킬 품질 향상

### 5.3 스킬 등록 및 활성화

1. 스킬 생성 대화에서 `스킬 저장` 버튼 클릭
2. `사용자 지정 → 스킬` 메뉴에서 등록 확인
3. 토글로 스킬 활성화/비활성화 관리
4. 이후 채팅에서 트리거 키워드 포함 문장 입력 시 자동 호출

## 6. 핵심 개념 요약표

| 개념 | 한 줄 정의 |
|---|---|
| **Artifacts** | 대화창 옆 패널에 렌더링되는 독립 결과물 |
| **Extended Thinking** | 응답 전 내부 사고 공간을 분리하는 추론 메커니즘 |
| **Project** | 관련 대화·파일·지침을 묶는 지속적 작업 공간 |
| **Persistent Context** | 프로젝트 내 모든 대화에 공유되는 system prompt |
| **Skill** | 반복 작업용 구조화 지침 묶음 (SKILL.md 기반) |
| **SKILL.md** | YAML 메타데이터(트리거) + 마크다운 본문(절차)으로 구성된 핵심 파일 |
| **Connector** | Claude가 외부 서비스/데이터에 접근하는 연결 통로 |
| **Hallucination** | LLM이 사실이 아닌 내용을 사실처럼 출력하는 현상 |
