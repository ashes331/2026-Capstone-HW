## 4. 대화 질문 설계 (프롬프트 엔지니어링)

### 4.1 정의와 중요성

**프롬프트 엔지니어링** = 생성형 AI와의 대화에서 원하는 결과를 얻기 위해 질문을 설계하고 최적화하는 과정. "대화 질문 설계" 또는 "대화 질문 최적화"라고도 부른다.

| 구성 요소 | 설명 |
|----------|------|
| 질문 설계 | 효과적인 질문을 만들기 위한 구조화된 접근 방식 |
| 최적화 기술 | AI 응답을 개선하기 위한 전략적 조정 |
| 사용자 의도 | 목표와 목적을 명확하게 전달 |
| AI 활용 | AI의 능력을 최대한 활용 |
| 성능 극대화 | 정확하고 유용한 정보 제공을 위한 AI 능력 극대화 |

**중요성:** 사용자의 질문 방식에 따라 답변의 정확도와 효율성이 달라진다. AI에게 역할을 부여하거나 특정 상황을 설정하고, 답변에 피드백을 제공함으로써 성능 향상에 기여한다.

---

### 4.2 단계별로 구체적으로 질문하기

**핵심 기법: "Let's think step by step"** 추가

복잡한 질문일수록 단계를 나누어 접근하면 AI가 더 명확하고 구체적인 정보를 제공한다.

**예시 — 서울 여행 계획:**

| 단계 | 질문 |
|------|------|
| 1단계 | 여행 목적을 정해주세요 |
| 2단계 | 출발지와 교통 수단은 무엇인가요? |
| 3단계 | 세부 일정을 어떻게 구성할까요? |

> **Before:** "서울 여행을 계획하고 싶어요"  
> **After:** 위 단계 + `Let's think step by step` 추가 → 구체적 시간표, 교통편, 맛집까지 포함된 상세 일정 생성

**환경 문제 예시 — 3단계 분해:**
- 1단계: 문제 정의
- 2단계: 원인 분석
- 3단계: 해결책 제안

## 5. 대화 질문의 구성 요소

### 5.1 목적 · 맥락 · 역할

| 요소 | 설명 | 나쁜 예 → 좋은 예 |
|------|------|-----------------|
| **목적 (Objective)** | AI 활용 성공 여부를 결정짓는 핵심. 무엇을 얻고자 하는지 구체적으로 정의 | "한국 역사 알려줘" → "조선시대 임진왜란의 배경과 결과를 요약해줘" |
| **맥락 (Context)** | 질문에 필요한 배경과 상황 정보 | "이 책에 대해 설명해줘" → "베스트셀러 '세상의 마지막 기차역'의 줄거리와 등장인물 소개해줘" |
| **역할 (Role/Persona)** | AI 모델이 특정 상황에서 수행해야 하는 기능 정의 | "임진왜란 설명해줘" → "네가 역사 선생님이라고 생각하고 임진왜란 설명해줘" |

### 5.2 청중 · 예시 · 형식

| 요소 | 설명 | 예시 |
|------|------|------|
| **청중 (Audience)** | 질문 대상자의 수준과 필요에 맞춰 프롬프트 설계 | "초등학교 5학년 수준", "역사학 전공 대학생 대상" |
| **예시 (Examples)** | AI에게 원하는 결과물의 형태·내용·스타일을 명확히 보여주는 핵심 요소 | "한국의 유명한 시인" → "윤동주의 '서시'처럼 감성적인 시를 쓰는 시인 3명 소개해줘" |
| **형식 (Format)** | AI가 답변을 생성하는 구조나 형식 지정 | "표 형식으로 정리해줘", "단계별로 설명해줘", "코드 예시 포함해줘" |

### 5.3 프롬프트에서 활용하는 주요 문자 기호

| 기호 | 용도 | 질문 예시 |
|------|------|---------|
| `-` 하이픈 | 목록의 각 항목 표시 | `- 주제 소개` / `- 주요 내용` / `- 요약` |
| `""` 따옴표 | 특정 단어·문구 강조, 정확한 인식 필요 텍스트 | `"인공지능"의 정의를 설명해주세요` |
| `:` 콜론 | 목록이나 설명 시작 | `이름: [이름], 나이: [나이]` |
| `*` 별표 | 목록 표시 또는 특정 내용 강조 | `*데이터 과학*의 주요 분야를 설명해주세요` |
| `[]` 대괄호 | 추가 정보나 선택 사항 표시 | `[사용자 입력]을 받아서 환영 메시지 출력` |
| `{}` 중괄호 | 특정 조건 명시 | `{제목}: "인공지능의 현재와 미래"` |
| ` ``` ` 백틱 3개 | 코드 블록 표시, 프롬프트 내 코드·명령어 구분 | ` ```python def add(a,b): return a+b``` ` |
| `#` 해시 | Markdown 제목/섹션 구분, 프롬프트 구조화 | `# 서론` / `## 본론` / `### 결론` |

---

### 5.4 대화 질문 설계의 10가지 구성 요소

| 유형 | 설명 | 예시 |
|------|------|------|
| **목적 (Objective)** | 글의 목적을 명확히 설정 | "전기차 구매 시 고려해야 할 사항을 알려줘" |
| **상황 (Context)** | 배경 정보, 상황, 추가 자료 설명 | "이 상황의 배경, 목표, 고려해야 할 점, 제약 사항을 포함해 설명해줘" |
| **역할 (Role, Persona)** | ChatGPT의 역할 또는 관점 지정 | "너는 대학 교수야. 마케팅 전문가라 생각하고 답해줘" |
| **청중 (Audience)** | 대상이 되는 목표 독자를 구체적으로 명시 | "초등학교 5학년 수준, 대학 신입생, 경영진 대상" |
| **예시 (Examples)** | 원하는 형식이나 스타일의 예시 제공 | "'긴 글 → 핵심 요약' 형태로 만들어줘" |
| **형식 (Format)** | 결과물의 구조나 형식 지정 | "문단으로 정리해줘, 표로 정리해줘" |
| **키워드 (Keywords)** | 반드시 포함해야 할 단어나 개념 지정 | "반복적 접근, 신속한 엔지니어링 포함해줘" |
| **제한 (Limitations)** | 답변의 길이, 단어 수, 문장 수 등의 제한 사항 | "200단어 이내, 3문장으로 작성, 예시 5개 포함" |
| **어조 (Tone)** | 글의 분위기와 스타일 지정 | "친절하게 설명해줘, 전문가의 시각에서 작성해줘" |
| **부정 (Negation)** | 피해야 할 내용 명시 | "기술 용어를 남발하지 말고, 너무 어렵게 설명하지 마" |

**실전 예시 — 10가지 요소를 모두 담은 프롬프트:**

```
교육 전문가의 역할로, 성인 학습자를 대상으로 하는 '시간 관리 전략'에 대해 설명해줘
- 자연스럽고 동기를 부여할 수 있는 어조를 사용하고
- 3가지 전략을 한 문단 씩 작성하며, 각 문단은 5문장 이내로 유지해줘
- 단, 지나치게 이론적인 내용은 포함하지 말고 실생활 예제를 포함해줘
- 예시: 아침 10분을 활용해 하루 계획을 세워보세요.
```

## 6. 효율적인 대화 질문: 퓨샷 프롬프팅

### 6.1 퓨샷 러닝 (Few-shot Learning, FSL)

AI 모델이 **제한된 수의 예시만으로** 새로운 개념이나 작업을 학습하는 기술. 기존 ML 모델이 방대한 데이터를 필요로 하는 것과 달리, 퓨샷 러닝은 적은 예시로도 높은 학습 효율을 보인다.

**활용 상황:**

| 상황 | 설명 |
|------|------|
| 데이터 부족 | 새로운 분야나 희귀한 데이터의 경우 학습 데이터 확보가 어려울 때 |
| 빠른 적응 | 새로운 환경이나 작업에 신속하게 적응해야 할 때 |
| 개인화 | 개인의 취향·특성에 맞춰 모델을 학습시킬 때 |

### 6.2 프롬프팅 유형 비교

| 유형 | 정의 | 특징 |
|------|------|------|
| **Zero-shot (ZSP)** | 예시 없이 바로 원하는 결과를 요청 | AI의 사전 지식에 의존. 단순하고 빠름 |
| **One-shot (OSP)** | 예시 1개를 제공하여 결과 유도 | Zero-shot보다 정확도 높음. 간단한 패턴 학습에 효과적 |
| **Two-shot (TSP)** | 예시 2개를 제공하여 결과 유도 | 패턴을 더 명확히 학습. One-shot보다 일관성 향상 |
| **Few-shot** | 여러 개의 예시를 제공하여 결과 유도 | 복잡한 패턴·규칙 학습 및 다양한 맥락 이해에 효과적 |

### 6.3 프롬프팅 유형별 질문 사례

**Zero-shot 예시:**
```
머신러닝과 딥러닝의 차이점은?
```

**One-shot 예시:**
```
Q: "머신러닝이란?" A: "데이터를 학습하여 패턴을 인식하는 기술입니다."
Q: "딥러닝이란?" A:
```

**Two-shot 예시:**
```
Q: "머신러닝이란?" A: "데이터 학습 기술입니다."
Q: "딥러닝이란?" A: "신경망을 이용한 학습 기법입니다."
Q: "강화학습이란?" A:
```

**Few-shot (감정 분류) 예시:**
```
다음은 예시입니다.
- [긍정]: '선물을 받아서 기분이 좋아요.'
- [부정]: '숙제를 못 해서 속상해요.'

다음 문장의 감정을 긍정 또는 부정으로 분류해 주세요: '오늘 옷을 사서 돈을 많이 지출했어요.'
```
