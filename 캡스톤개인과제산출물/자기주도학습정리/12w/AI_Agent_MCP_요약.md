## 1. AI Agent 개요

### 에이전트(Agent)란?
- 환경을 **지각(Perceive)** 하고, 그 지각에 따라 **행동(Act)** 하는 모든 것

### AI Agent
- AI 기술로 환경을 지각하고, 합리적 판단을 통해 **목표를 달성하는 시스템**

| 구분 | 인간 | AI Agent |
|------|------|----------|
| 지각 | 오감 (시각·청각·후각 등) | 센서 (카메라·마이크·온도센서 등) |
| 행동 | 근육·언어·표정 등 | 액추에이터 (모터·디스플레이·스피커 등) |
| 목표 | 생존·행복·학습 등 | 주어진 목표 달성 (경로 탐색, 추천 등) |

### LLM 단독의 한계
- LLM은 학습된 데이터에 **갇혀 있음**
- 텍스트 생성은 가능하지만, **실제 행동**(메일 발송, 파일 수정, 코드 실행 등)은 직접 불가
- → 외부 도구(Tools)와의 연결로 한계를 극복

## 2. MCP(Model Context Protocol) 개요

### MCP란?
- AI 모델이 외부 데이터 소스·도구와 **안전하고 표준화된 방식**으로 통신하는 오픈소스 프로토콜
- Anthropic(Claude 개발사)이 주도 개발
- **"AI 생태계의 USB-C 포트"** — 하나의 표준으로 모든 도구에 연결

### MCP 이전 vs 이후

| 구분 | 과거 (N:N) | 현재 (1:N) |
|------|-----------|-----------|
| 방식 | 도구마다 개별 맞춤형 API 개발 필요 | 단 하나의 범용 프로토콜로 모든 도구 통제 |
| 비유 | 규격이 다른 케이블들 | USB-C 하나로 모두 연결 |

### MCP의 3대 특성

| 특성 | 설명 |
|------|------|
| **표준화** | MCP 서버를 한 번만 구축하면 MCP 지원 AI 에이전트 모두와 연결 가능 |
| **보안** | 사용자가 권한을 직접 통제하며 안전하게 데이터 교환 |
| **확장성** | 로컬 파일·사내 DB·외부 API 등 무한한 도구 연결 가능 |

### MCP 아키텍처 구성 요소

```
MCP Host (Claude Desktop, Cursor IDE, Manus AI ...)
    └── MCP Client  ←→  MCP Server ── Remote Services (Slack, Gmail ...)
                         MCP Server ── Local Data Sources (파일, DB ...)
```

| 구성 요소 | 역할 |
|----------|------|
| **MCP Host** | AI 에이전트 프로그램이 실행되는 환경 |
| **MCP Client** | 호스트 내부에 존재하며 서버와 통신 담당 |
| **MCP Server** | 외부 도구/데이터 소스를 감싸고 있는 프로그램, 클라이언트 요청을 실제로 실행하고 결과 반환 |

## 3. 클로드 커넥터 개요

### 커넥터란?
- **"대화만 하는 AI"** → **"내 업무 도구를 실제로 다루는 AI"** 로 확장하는 장치
- 외부 서비스(Notion, Gmail, Google Calendar 등)의 데이터와 기능에 접근하는 연결층
- **노코드(No-code)** 솔루션 — 복잡한 설정 없이 클릭 몇 번으로 연결

### 커넥터의 두 가지 연결 방식

| 방식 | 대상 | 특징 |
|------|------|------|
| **공식 인티그레이션** | 일반 사용자 | OAuth 기반, 간편 연결, 안정적 서버 사용 |
| **MCP** | 개발자 | 오픈소스 표준, 로컬 파일·커스텀 워크플로우 구축 최적화 |

### OAuth(Open Authorization)
> 인터넷 사용자가 **비밀번호를 노출하지 않고**, 자신의 정보가 저장된 서비스(Google, Facebook 등)의 권한을 다른 애플리케이션에 안전하게 부여하는 **개방형 표준 인증 프로토콜**

### 커넥터가 필요한 이유

1. **정확도 향상** — 여러 앱을 오가며 실제 데이터를 기반으로 답변
2. **업무 자동화** — 반복적 업무를 하나의 흐름으로 처리
   - 예: 노션 회의록 생성 → Gmail 초안 작성 → Google Calendar 일정 등록

### 도구 권한 설정

| 설정 | 동작 |
|------|------|
| **항상 허용** | Claude가 승인 없이 도구를 자율 실행 |
| **승인 필요** | 도구 실행 전 사용자 확인 요청 |

## 4. 커넥터 활용 사례

### 사례 1 — Gmail: 메일 자동 분류 및 답장 초안
```
프롬프트 예시:
"오늘 온 gmail에서 중요한 메일부터 정리해 줘"
"최근 5일간 메일 중 중요 메일 요약해줘"
"학생 문의 메일에 답장 초안 작성해 줘"
```
- 결과: 중요도 순 메일 요약 테이블 생성, 답장 초안 Gmail 임시보관함 저장
- **주의**: 메일 직접 전송(`send_email`)은 현재 쓰기/삭제 도구의 **승인 필요** 권한 필요

---

### 사례 2 — Gmail + Google Calendar: 카드 납부 일정 자동 등록
```
프롬프트 예시:
"내 Gmail에서 최근 롯데카드 청구서 이메일을 검색해줘.
 이메일에서 청구 금액과 납부 마감일을 추출한 뒤,
 Google 캘린더의 해당 마감일에 '롯데카드 납부: [금액]원' 형식으로 일정을 등록해줘."
```
- **Claude 커넥터**: 본문 텍스트 기반 정보 추출 후 캘린더 등록 가능
- **한계**: 첨부파일(PDF) 바이너리 직접 추출은 현재 미지원 (`get_thread`가 본문 텍스트만 반환)
- **Manus 비교**: 브라우저를 직접 조작해 첨부파일까지 처리하지만 시간이 오래 걸림

---

### 사례 3 — Google Drive: 채팅 결과 저장
```
프롬프트 예시:
"위 내용을 내 구글 드라이브에 \claude\connector 폴더를 만들어 저장해줘"
```
- Claude가 자동으로 폴더 구조 생성 및 Google Docs로 저장

#### Google Drive 주요 활용 패턴

| 카테고리 | 예시 |
|----------|------|
| 파일 탐색/검색 | "최근 수정한 파일 5개 보여줘", "'머신러닝' 키워드 포함 문서 검색해줘" |
| 문서 읽기/분석 | "Google Docs 링크 내용 요약해줘", "논문 초안 읽고 피드백 줘" |
| 문서 작성/업로드 | "이 내용으로 새 Google Doc 만들어서 드라이브에 올려줘" |
| 다른 커넥터 조합 | "드라이브 강의자료 읽어서 Gmail로 학생들에게 요약 발송해줘" |

---

### 사례 4 — Notion: 질의 결과 저장
```
프롬프트 예시:
"위 내용을 나의 노션에 페이지 [2026 클로드 추가된 기능]를 만들어 저장해 줘"
```
- Notion MCP 연결 시 워크스페이스 선택 후 OAuth 권한 부여 필요
- Claude가 자동으로 페이지 생성 및 구조화된 콘텐츠 작성

---

### 사례 5 — KakaoTalk: PlayMCP 연결
```
연결 순서:
1. https://playmcp.kakao.com 접속 → 카카오 계정 로그인
2. '카카오톡 나와의 채팅방' MCP 검색 → 도구함에 추가
3. Claude 커넥터에서 'PlayMCP' 검색 → 연결 (인증 3~4분 소요)
4. 채팅창에서 카톡 발송 요청
```
- Claude가 카카오톡 나와의 채팅방(MemoChat)으로 메시지 전송 가능

## 5. Claude in Chrome (브라우저 에이전트)

### 설치 방법
1. Chrome 웹 스토어에서 "Claude" 검색 → **Add to Chrome**
2. 확장 프로그램 아이콘 고정 (툴바 핀)
3. Claude 계정 로그인 → 권한 허용

### 주요 기능

| 기능 | 설명 |
|------|------|
| **웹페이지 요약·정리** | 현재 열린 페이지 내용 즉시 분석 |
| **웹 검색 후 접속** | 질의에 맞는 사이트 자동 탐색 및 정보 추출 |
| **워크플로우 자동화** | 버튼 클릭, 폼 작성 등 반복 작업 자동화 |
| **프레젠테이션 생성** | Gamma 등 외부 도구와 연동해 PPT 자동 생성 |
| **Claude Code 통합** | 코드 작성 후 브라우저에서 바로 테스트·디버깅 |

### 보안 제한
- **금융 사이트** 등 민감 카테고리(sensitive site)는 접근 제한
- 브라우저 에이전트의 **프롬프트 인젝션 방어** 적용

## 6. 핵심 개념 정리

### 용어 요약

| 용어 | 정의 |
|------|------|
| **AI Agent** | 환경을 지각하고 합리적 판단으로 목표를 달성하는 AI 시스템 |
| **MCP** | AI 모델과 외부 도구 간 표준화된 통신 프로토콜 (Anthropic 주도) |
| **MCP Host** | AI 에이전트가 실행되는 환경 (Claude Desktop, Cursor 등) |
| **MCP Client** | 호스트 내부에서 서버와 통신하는 컴포넌트 |
| **MCP Server** | 외부 도구/서비스를 감싸고 실제 작업을 수행하는 프로그램 |
| **커넥터** | Claude와 외부 서비스를 연결하는 노코드 통합 레이어 |
| **OAuth** | 비밀번호 노출 없이 외부 서비스 권한을 위임하는 표준 인증 프로토콜 |
| **PlayMCP** | 카카오의 MCP 허브 플랫폼 (카카오톡·캘린더·지도 등 연결) |

### 아키텍처 흐름 요약

```
사용자 요청
    ↓
Claude (LLM / AI Agent)
    ↓
커넥터 / MCP Client
    ↓
MCP Server
    ↓
외부 도구 (Gmail, Notion, Google Drive, KakaoTalk ...)
    ↓
결과 반환 → Claude가 가공 → 사용자에게 응답
```
